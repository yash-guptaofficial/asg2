DESIGN PATTERNS & PRINCIPLES
═══════════════════════════════════════════════════════════════════

✓ Page Object Model (POM)
  - Separate page logic from test logic
  - Reusable page objects
  - Centralized locators

✓ DRY (Don't Repeat Yourself)
  - Base class with common methods
  - Locators in single file
  - Shared utilities

✓ Clean Code
  - Meaningful method names
  - Single responsibility
  - Proper error handling

✓ Maintainability
  - Easy to add new tests
  - Easy to update locators
  - Clear separation of concerns


KEY FEATURES
═══════════════════════════════════════════════════════════════════

1. WAIT STRATEGIES
   - Explicit WebDriverWait
   - 10-second default timeout
   - Visibility checks before interaction

2. ERROR HANDLING
   - Try-except for optional elements
   - Meaningful assertion messages
   - Exception logging

3. ELEMENT INTERACTION
   - Safe click with wait
   - Clear before sendKeys
   - Scroll to element
   - JavaScript execution fallback

4. ASSERTIONS
   - Element visibility
   - Cart count verification
   - Product name matching
   - Success message validation
